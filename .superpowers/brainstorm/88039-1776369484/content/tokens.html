Design Section 1: Token System

Spacing, elevation, and shape tokens that replace all ad-hoc inline values

Spacing Tokens — RealmsSpacing

Every inline padding/margin snaps to one of these 7 values. Current inconsistency (14/16/18/20/22dp margins) collapses to l = 16.dp everywhere.

xxs
2.dp pip gaps, wealth bar segments
xs
4.dp tight inner gaps, section cap padding
s
8.dp row spacing, chip gaps, default element spacing
m
12.dp compact card padding, list item padding
l
16.dp universal screen margin, standard card padding
xl
20.dp section-to-section vertical gaps
xxl
24.dp dialog/overlay internal padding

Elevation Tokens — RealmsElevation

Three named levels replacing 6 different ad-hoc values (1/2/3/4/6/8/10dp).

low = 2.dp
Top bar, input bar, player bubble
medium = 6.dp
Overlays, rest dialog, map panel
high = 10.dp
Pre-roll dialog, critical overlays

Shape Enforcement

No new values — RealmsShapes is already correct. The change is how they're referenced: inline RoundedCornerShape(Ndp) becomes MaterialTheme.shapes.*.

Current (inline) Becomes Used for Preview
RoundedCornerShape(6.dp) shapes.extraSmall Badges, level chip, tiny indicators
RoundedCornerShape(10.dp) shapes.small Filter tabs, chips, compact buttons
RoundedCornerShape(14.dp) shapes.medium Cards, list items, full-width buttons
RoundedCornerShape(20.dp) shapes.large Message bubbles, dialog cards
RoundedCornerShape(28.dp) shapes.extraLarge Bottom sheet tops, FAB

ModalBottomSheet currently overrides its top corners with RoundedCornerShape(topStart = 28.dp, topEnd = 28.dp) — this becomes BottomSheetDefaults.ExpandedShape (M3 default).
