RealmsCard — the workhorse

Wraps M3 Card / OutlinedCard / ElevatedCard behind a single API with game-specific defaults.

FILLED (default)
RealmsCard { }
Uses surfaceVariant.copy(0.45f)
Shape: shapes.medium
Most list items, detail cards
OUTLINED
RealmsCard(outlined = true, accentColor = ...) { }
Border from accentColor param
Shape: shapes.medium
Quests, selected spells, equipped items
ELEVATED
RealmsCard(elevation = RealmsElevation.medium) { }
Uses tonalElevation
Shape: shapes.large
Overlays, dialogs, detail panels
@Composable
fun RealmsCard(
  modifier: Modifier = Modifier,
  onClick: (() -> Unit)? = null,
  shape: Shape = MaterialTheme.shapes.medium,
  outlined: Boolean = false,
  accentColor: Color? = null, // border + selected tint
  selected: Boolean = false,
  elevation: Dp = 0.dp,
  content: @Composable ColumnScope.() -> Unit
)

When selected = true and accentColor is set, the card tints its background with accentColor.copy(0.1f) and shows a full-opacity border — matching the current spell/inventory selection pattern. When onClick is non-null, the card is clickable.

RealmsProgressBar

Unifies 4 separate progress bar implementations into one. Wraps M3 LinearProgressIndicator.

HP — dynamic color (red/yellow/green) 72/100
XP — fixed secondary 450/1000
Morality — dynamic color by sign -30
@Composable
fun RealmsProgressBar(
  progress: Float,
  color: Color,
  modifier: Modifier = Modifier,
  height: Dp = 6.dp,
  trackColor: Color = MaterialTheme.colorScheme.surfaceVariant
)

Callers still decide the color logic (HP tier, morality sign, fixed secondary). The component just ensures consistent height, corner radius, and track color. Replaces both LinearProgressIndicator calls and the custom Box-in-Box bar in CombatHud.
