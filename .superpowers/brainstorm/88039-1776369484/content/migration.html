Design Section 4: Screen Migration Strategy

How each screen/file group changes once the foundation layer is in place

Migration Order

Foundation first, then files grouped by dependency. Each group can be done in parallel.

PHASE 0
Foundation
Add RealmsSpacing + RealmsElevation to ui/theme/
Create ui/components/ with all 8 shared composables
Add npcPalette + scrimOverlay to RealmsExtendedColors
Delete PanelShared.kt (contents moved to components/)
~6 new files, 1 deleted, 2 modified
PHASE 1
Panels (9 files) — parallel
All panels import from ui/components/ instead of PanelShared
Replace Surface-as-card → RealmsCard
Replace inline shapes → MaterialTheme.shapes.*
Replace inline padding → RealmsSpacing.*
Replace inline section headers → SectionHeader
InventoryPage, QuestsPage, PartyPage, LorePage, NpcJournalPage, CurrencyPage, SpellsPage, StatsPage — each is independent
PHASE 2
Game screen (10 files) — parallel
Same mechanical replacements as Phase 1
MessageBubbles.kt: move npcColorPalette → RealmsTheme.colors.npcPalette
TopBar.kt: HP/XP bars → RealmsProgressBar
CombatHud.kt: Box bar → RealmsProgressBar
GameDialogs.kt: elevation → RealmsElevation.high
GameScreen, MessageBubbles, NarrationBlock, GameDialogs, GameOverlays, ChatInput, TopBar, CombatHud, ChatFeed, BottomNav
PHASE 3
Overlays (3 files) — parallel
Overlays.kt: elevation → tokens, shapes → tokens. Campfire Canvas colors stay inline.
ShopOverlay.kt: Surface cards → RealmsCard, padding → spacing tokens
TargetPromptDialog.kt: shapes + padding → tokens
3 files, all independent
PHASE 4
Setup + Map (5 files) — parallel
CharacterCreationScreen.kt: delete private SectionHeader, use shared. Shapes + padding → tokens. Button shapes → shapes.medium.
TitleScreen.kt: graveyard gradient → theme colors. Shapes + padding → tokens.
DeathScreen.kt: button shapes → tokens, section headers → SectionHeader.
ApiSetupScreen.kt: purple highlight → secondaryContainer. Shapes → tokens.
WorldMapScreen.kt: elevation → tokens, scrim → scrimOverlay. Road Canvas colors stay.
5 files, all independent

What Changes Per File (mechanical checklist)

Every file in Phases 1–4 gets the same set of mechanical replacements:

1 RoundedCornerShape(N.dp) → MaterialTheme.shapes.{extraSmall|small|medium|large|extraLarge}
2 padding(N.dp) → padding(RealmsSpacing.{xxs|xs|s|m|l|xl|xxl})
3 tonalElevation = N.dp → tonalElevation = RealmsElevation.{low|medium|high}
4 Surface(shape, color, border) { Column { ... } } → RealmsCard(...) { ... }
5 Color(0x...) / Color.Black.copy(...) → theme token or RealmsTheme.colors.*
6 Remove unused import ...RoundedCornerShape and import ...dp where possible

What Does NOT Change

Navigation architecture — manual when(screen) state machine stays
Typography — Cinzel + Crimson Text identity stays, RealmsTypography unchanged
Canvas art — campfire flames, map terrain, sky gradients keep inline colors
Game logic, ViewModel, data layer — zero changes
Message bubble composables — layout stays, only colors change to theme refs
